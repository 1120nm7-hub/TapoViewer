# TapoのRTSPを使って、Windows用の自作カメラビューアを作った話

## きっかけ

TP-Link製の見守りカメラ「Tapo」は、公式アプリ(スマホ)からは複数台の映像を快適に確認できますが、Windows PCから同じように見る手段が公式には用意されていません。ブラウザ経由のクラウドビューアはあるものの、遅延や画質、複数台同時表示の面で不満があったため、カメラ側で公開されているRTSP機能を使い、Windows用のビューアを自作しました。

作ったものはGitHubで公開しています。

- リポジトリ: https://github.com/1120nm7-hub/TapoViewer

> 本記事・本ツールはTP-Link社およびTapoブランドとは無関係の個人プロジェクトです。「Tapo」はTP-Link Systems Inc.の商標です。カメラが公式に提供しているローカルRTSP配信を利用しているだけで、Tapoアプリやクラウド機能を代替・改変するものではありません。

## できること

| 機能 | 内容 |
|---|---|
| 複数台ライブ表示 | グリッド/縦並び/横並びの3レイアウトに対応、自動再接続あり |
| HD/SD切替 | カメラごとにワンクリックで画質(ストリーム)切替 |
| 録画 | ffmpegのストリームコピーで無劣化保存(CPU負荷ほぼゼロ) |
| 音声再生 | カメラのマイク音声をPC側で再生(1台ずつ切替) |
| スナップショット | JPEG保存 |
| 映像回転・タイムスタンプOSD | 設置角度が斜めになりがちな環境向け。OSDはTapo本体の時刻表示と被らないようデフォルトOFF・表示時は右下に配置 |
| カードサイズ切替(S/M/L) | 台数や画面サイズに応じて一覧の見え方を調整 |
| 保存先のカスタマイズ | 録画・スナップショットの保存フォルダをそれぞれ変更可能 |

## 使っている技術

- **表示・UI**: Python + [customtkinter](https://github.com/TomSchimansky/CustomTkinter)
- **映像取得**: OpenCV(`cv2.VideoCapture` + `CAP_FFMPEG`、`rtsp_transport=tcp`で安定化)
- **録画**: `ffmpeg -c:v copy`でストリームをそのままコンテナ化(再エンコードなし)。音声のみカメラ側のPCM/ALawがMP4非対応なのでAACに変換
- **音声再生**: ffmpegでPCMにデコードし、[sounddevice](https://python-sounddevice.readthedocs.io/)で出力
- **配布**: PyInstallerで単一exe化(ffmpegバイナリは`imageio-ffmpeg`経由で同梱、追加インストール不要)

## つまずいたポイント

- **RTSPの再接続**: Wi-Fi環境だと数分〜数十分に一度接続が切れることがあったため、読み取り失敗が続いたら自動で再接続するループを実装
- **録画停止時の不正終了**: ffmpegを強制killするとMP4のインデックス(moov atom)が書き込まれず再生できないファイルができてしまう問題があったため、標準入力に`q`を送って正常終了させるようにした
- **音声と映像の帯域**: 複数台同時に高画質ストリームを取得すると帯域を圧迫するため、音声だけは軽量な`stream2`(SD)側から取得している

## 事前準備(カメラ側)

Tapoアプリで「カメラのアカウント」を有効化しておく必要があります(スマホのTapoアプリのログインアカウントとは別物です)。

1. Tapoアプリでカメラを開く
2. 設定(歯車) → 詳細設定 → カメラのアカウント
3. ユーザー名・パスワードを設定して有効化
4. カメラのIPアドレスを確認(ルーターでのIP固定を推奨)

## セットアップ

一番手軽なのは [Releases](https://github.com/1120nm7-hub/TapoViewer/releases/latest) から `TapoViewer.exe` を直接ダウンロードする方法です(Pythonのインストール不要、ffmpeg同梱済み)。

ソースから自分でビルドしたい場合は以下の手順です。

1. Python 3.10以降をインストール(PATHに追加するオプションを忘れずに)
2. リポジトリを取得し、`build_exe.bat`を実行
3. `dist\TapoViewer.exe`をお好きな場所へ

詳細は[README](https://github.com/1120nm7-hub/TapoViewer#readme)を参照してください。

## プライバシーについて

映像・音声はすべて同一LAN内のカメラ⇔PC間のRTSP通信のみで完結し、外部サーバーには一切送信されません。カメラの認証情報は`%APPDATA%\TapoViewer\`にローカル保存されます(平文のため取り扱い注意)。テレメトリや広告SDKの類は組み込んでいません。
